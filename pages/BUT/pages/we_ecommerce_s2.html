SAE2.04 : Développement d'un site e-commerce

Description du projet

En partant d’un cahier des charges, il fallait réaliser et étudier une base de données. À partir d’un jeu d’essais, il fallait proposer d'une visualisation des informations permettant d’apporter une analyse à l’entreprise. Cette SAÉ permet une première approche complète des aspects de conception, implémentation, administration et exploitation d’une base de données.

Nous devions donc étudier les données et visualiser les informations à partir d'un projet python et une base de donnée, crée une modèle conceptuel de base de donnée, faire une présentation orale des résulats en anglais, un script de création de base de données et enfin une interface pour ajouter/modifier/supprimer des données et visualiser ces informations.

Ma contribution dans ce projet

Mon travail au début du projet était d'adapter la partie sécurité pour notre projet, les mots de passe des comptes utilisateurs sont cryptés par exemple (la plupart de code était donné et expliqué en TD). Une fois le panier et le code adapté à notre projet (affichage des peintures dans la boutique et non des articles), mon travail fut de trouver le code pour calculer les sommes d'argent dans le panier et m'occuper de toute la partie commande:

  • Passer une commande une fois les articles sélectionnés dans le panier
  • Affichage dans une autre page, toutes les commandes effectuées par le client
  • Affichage des détails de chaque commande
  • Donner à l'admin le pouvoir d'expédier les commandes de tous les clients ("en traitement" par défaut)


Le rapport ci-dessous a été écrit par mon collègue Aurélien Guillou, j'ai simplement participé à la réalisation du MCD et à la mise en page du rapport dans cette partie.

À partir d'un modèle conceptuel de données, j'ai permis l'affichage et le bon fonctionnement du site: l'ajout de nouveaux articles en tant qu'administrateur, l'affichage de graphiques visuels sur les nombreuses données comme la quantité de commentaires postés pour chaque article etc..

J'ai optimisé le modèle au besoin de la demande énoncé dans le cahier des charges et coordonner une équipe informatique pour satisfaire cette demande dans les meilleurs délais et coûts. Ces 3 compétences sont dans la bonne voie et je travaillerai davantage pour les consolider.

Retour réflexif

Le projet c'est très bien déroulé, un point qu'il faut améliorer de mon côté est ma capacité à diriger une équipe, le travail était assez inégal entre les membres du groupe, j'aurais peut-être pu mieux découper les tâches. Mais certains ont plus de difficultés que d'autres et nous avions des délais relativement courts donc ce résultat était probablement prévisible. L'important est que chacun s'est améliorer à son rythme lors de ce projet, d'un point de vue pédagogique.
